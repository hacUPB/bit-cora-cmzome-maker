## CICLO FETCH–DECODE–EXECUTE
- Es la forma en que el computador ejecuta instrucciones
- Primero toma la instrucción, luego la entiende y después la ejecuta
- El Program Counter indica cuál instrucción sigue
- El PC va avanzando mientras el programa se ejecuta

## INSTRUCCIÓN-A VS INSTRUCCIÓN-C
- La instrucción-A empieza con @
- Sirve para cargar valores o direcciones
- Ejemplo: @5
- La instrucción-C se usa para hacer operaciones
- Ejemplo: D=M

## COMPONENTES DEL COMPUTADOR HACK
- El registro A guarda direcciones o valores
- El registro D se usa para hacer operaciones
- La ALU se encarga de los cálculos

## SALTOS CONDICIONALES
- Se hacen usando el valor que queda en D
- Dependiendo de ese valor el programa salta
- Ejemplo: D;JGT salta si D es mayor que cero

## IMPLEMENTACIÓN DE LOOPS
- Se usan etiquetas para repetir instrucciones
- El programa vuelve al inicio del ciclo
- El loop termina cuando se cumple una condición

## DIFERENCIA ENTRE D=M Y M=D
- D=M lee un valor desde memoria
- M=D guarda un valor en memoria

## USO DE KBD Y SCREEN
- KBD sirve para leer lo que se presiona en el teclado
- SCREEN se usa para mostrar cosas en la pantalla
